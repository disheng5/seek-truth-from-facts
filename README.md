# 🧭 Seek Truth from Facts（实事求是）

**一套从毛泽东选集中提炼的哲学方法论工具，帮你在复杂局面中看清本质、做出判断、推动落地。**

不是政治课，不是心灵鸡汤。是一套经过战争与实践反复验证的**思维操作系统**——矛盾怎么拆、优先级怎么排、逆境怎么熬、团队怎么带、复盘怎么做，全有对应的框架。

> *"没有调查就没有发言权。"*
> *"你要知道梨子的滋味，你就得变革梨子，亲口吃一吃。"*

这些话你可能听过无数遍。但当它们被结构化成方法论，接入AI的分析能力，应用到你眼前的具体问题上——你会发现，老人家的智慧远比你以为的实用。

---

## 为什么需要这个 Skill？

大模型很擅长分析问题，但它的默认思维方式是"西方MBA式"的——SWOT、波特五力、OKR。这些当然有用，但面对真正复杂、模糊、充满矛盾的现实问题时，往往给你一堆"正确的废话"。

这个 Skill 给 AI 装了一套**东方哲学的思维内核**：

- 不是列pros and cons，而是**找主要矛盾**——解决一个，带动全局
- 不是做SWOT四象限，而是**判断矛盾性质**——内部矛盾靠协商，对抗性矛盾靠斗争
- 不是画完美路线图，而是**划分战略阶段**——防御期、相持期、反攻期各有打法
- 不是说"要做用户调研"，而是告诉你**"没有调查就没有发言权"具体怎么落地**

## 八大方法论

| # | 方法论 | 一句话 | 你什么时候需要它 |
|:---|:---|:---|:---|
| 1 | **矛盾分析法** | 找到那个"牛鼻子" | 问题太多不知道先解决哪个 |
| 2 | **实践认识论** | 别想了，先试 | 方案争论不休，谁也说服不了谁 |
| 3 | **持久战略法** | 熬得住，等得到 | 竞争对手太强、资源太少、压力太大 |
| 4 | **调查研究法** | 别猜，去看 | 拍脑袋做的决策总是翻车 |
| 5 | **群众路线法** | 答案在一线 | 制定的方案推不动、没人用 |
| 6 | **统一战线法** | 把朋友搞得多多的 | 跨团队协作推不动、资源整合难 |
| 7 | **独立自主法** | 核心的东西自己得有 | 自研还是外购？依赖别人靠谱吗？ |
| 8 | **批评与自我批评法** | 先从自己身上找原因 | 项目挂了怎么复盘、团队有分歧怎么办 |

## 真实场景示例

### 🤖 AI / 技术研发

> **"我们团队在做一个RAG项目，向量检索、知识图谱、传统搜索三条技术路线各有支持者，CTO让我两周内定方案，怎么决策？"**

→ 矛盾分析法：三条路线不是"哪个最先进"的问题，而是"哪个在你的数据特征和团队能力下ROI最高"的问题。先拆矛盾——是准确率vs延迟的矛盾？还是短期交付vs长期架构的矛盾？抓住主要矛盾再决策。

> **"大模型推理成本太高，老板要求把API调用成本砍一半，但产品经理说效果不能降。怎么办？"**

→ 持久战略法 + 实践认识论：这是典型的"以时间换空间"。阶段一（防御期）：先用prompt优化和缓存策略砍掉最容易砍的30%；阶段二（相持期）：做模型蒸馏或混合路由，小模型处理简单query；阶段三：量变到质变，推理框架升级。每个阶段设止损线，跑通了再进下一步。

> **"公司要不要自己训练基座大模型？现在用的是第三方API，但数据安全和定制化一直是痛点。"**

→ 独立自主法：先做能力盘点和依赖审计——你有多少高质量数据？算力储备够吗？团队有几个做过预训练的人？然后评估"断供风险"——如果API提供商明天涨价3倍或停服，你的Plan B是什么？核心结论往往不是"全部自研"或"全部外购"，而是"数据能力必须自主，推理层可以渐进替换"。

### 👥 团队管理 / 组织问题

> **"我带了一个15人的团队，里面有两个tech lead一直不对付，各带各的小团队，代码review互相挑刺，严重影响交付效率。"**

→ 批评与自我批评法 + 矛盾分析法：先判断矛盾性质——这是技术路线之争（非对抗性），还是已经变成了人际对立（趋向对抗性）？然后"团结—批评—团结"：你作为leader先自我批评，再分别1对1谈，最后建立议事规则。记住：开掉一个人是最后手段，建机制才是治本。

> **"新来的产品经理提了一堆需求，技术团队觉得不靠谱但又说不出具体哪里不对，两边互相看不顺眼。"**

→ 调查研究法 + 群众路线法：让产品去跟5个真实用户聊，让技术去看线上监控数据。"没有调查就没有发言权"——当双方都带着真实数据回来讨论时，90%的争论会自动消失。

### 📈 业务战略 / 职业发展

> **"我们是个20人的AI创业公司，刚拿了A轮。大厂刚发布了和我们功能类似的产品，免费的。怎么办？"**

→ 持久战略法 + 统一战线法："战略上藐视敌人，战术上重视敌人"。大厂做的是通用方案，你做垂直场景。现在是"战略防御期"——不要全面对抗，集中力量在你最强的3个客户场景里建根据地。同时找到可以联合的力量——同赛道非直接竞品、被大厂忽视的渠道商、对大厂方案不满意的客户。

> **"35岁了，技术能力还行但也没特别突出，管理岗竞争激烈，要不要跳槽？"**

→ 矛盾分析法 + 独立自主法：先找主要矛盾——你焦虑的根源是"收入天花板"还是"成长停滞"还是"安全感缺失"？不同矛盾对策完全不同。然后做能力盘点——你的不可替代性在哪里？哪些能力是"带得走的"？跳槽只是手段，核心是建立自己的能力壁垒。

## 使用方式

### 作为 Claude Code / Mira Skill

将 `SKILL.md` 放入你的 Skill 目录即可自动加载。

### 触发方式

不需要记住方法论名称，正常描述你的问题就行：

- *"帮我分析一下当前的竞争形势"*
- *"团队有分歧怎么办"*
- *"这个项目该怎么规划，资源不够"*
- *"我们应该自研还是外购"*
- *"项目失败了，怎么复盘才有用"*
- *"优先级排不出来，什么都重要"*

也可以直接指定框架：*"用矛盾分析法帮我拆一下这个问题"*

## 设计哲学

**不是给你答案，是给你一套拆问题的方式。**

这个 Skill 的每次输出都包含：

- **问题重定义**：你以为的问题往往不是真正的问题
- **方法论分析**：用结构化框架拆解，而不是凭感觉
- **可执行建议**：具体到"下一步做什么"，不说正确的废话
- **实践检验**：把结论变成可验证的假设——"实践是检验真理的唯一标准"
- **批判性反思**：诚实地指出分析的局限和可能的反驳

它会**敢说你不爱听的话**——好顾问不是只说好听话的。
