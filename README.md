# 🧭 Seek Truth from Facts（实事求是）

**一套从教员著作中提炼的哲学方法论工具，帮你在复杂局面中看清本质、做出判断、推动落地。**

不是政治课，不是心灵鸡汤。是一套经过实践反复验证的**思维操作系统**——矛盾怎么拆、优先级怎么排、逆境怎么熬、团队怎么带、复盘怎么做，全有对应的框架。

---

## 这套方法论的核心价值

**一个字：实。**

- **直击本质**：不绕弯子，从纷繁表象中找到那个"牛鼻子"。抓住主要矛盾，一切问题迎刃而解
- **阶段清晰**：任何复杂局面都能划分阶段——现在该守、该熬、还是该攻，各阶段打法完全不同
- **实践为王**：不相信任何未经验证的结论。每个分析都输出可验证假设，用事实说话
- **敢说真话**：好顾问不只说好听话。该指出的盲点一定指出，该泼的冷水一定泼
- **知行合一**：每条建议具体到"下一步做什么"，不说正确的废话

> *"一切结论产生于调查情况的末尾，而不是在它的先头。"*

## 八大方法论

| # | 方法论 | 一句话 | 你什么时候需要它 |
|:---|:---|:---|:---|
| 1 | **矛盾分析法** | 找到那个"牛鼻子" | 问题太多不知道先解决哪个 |
| 2 | **实践认识论** | 别想了，先试 | 方案争论不休，谁也说服不了谁 |
| 3 | **持久战略法** | 熬得住，等得到 | 竞争对手太强、资源太少、压力太大 |
| 4 | **调查研究法** | 别猜，去看 | 拍脑袋做的决策总是翻车 |
| 5 | **群众路线法** | 答案在一线 | 制定的方案推不动、没人用 |
| 6 | **统一战线法** | 把朋友搞得多多的 | 跨团队协作推不动、资源整合难 |
| 7 | **独立自主法** | 核心的东西自己得有 | 自研还是外购？依赖别人靠谱吗？ |
| 8 | **批评与自我批评法** | 先从自己身上找原因 | 项目挂了怎么复盘、团队有分歧怎么办 |

## 真实场景示例

### 🤖 AI / 技术研发

**"RAG项目，向量检索、知识图谱、传统搜索三条路线，CTO让我两周内定方案"**

> **主要矛盾**：不是"哪条路线最先进"，而是"哪条在你的数据特征和团队能力下两周内能跑通"。建议：用一周对三条路线做最小PoC，用真实query跑benchmark，数据说话。"你要知道梨子的滋味，你就得亲口吃一吃。"
>
> *教员语录：调查就像"十月怀胎"，解决问题就像"一朝分娩"。*

**"大模型推理成本太高，老板要砍一半，产品经理说效果不能降"**

> **阶段划分**：这不是一步到位的事。第一阶段（止血）：prompt精简+缓存命中，砍掉最容易的30%；第二阶段（相持）：模型路由，简单query走小模型；第三阶段（反攻）：蒸馏或微调专用模型。每阶段设止损线，跑通再进下一步。
>
> *教员语录：不打无准备之仗，不打无把握之仗。*

**"公司要不要自己训练基座大模型？用的是第三方API，数据安全和定制化是痛点"**

> **能力盘点**：你有多少高质量数据？算力够吗？团队有几个做过预训练的？**依赖审计**：API提供商明天涨价3倍或停服，Plan B是什么？结论往往不是全自研或全外购，而是"数据能力必须自主，推理层渐进替换"。
>
> *教员语录：我们的方针要放在自己力量的基点上，叫做自力更生。*

### 👥 团队管理

**"两个tech lead一直不对付，各带各的小团队，代码review互相挑刺"**

> **矛盾性质判定**：技术路线之争（非对抗性）还是人际对立（趋向对抗性）？非对抗性矛盾靠协商和建机制，不靠开人。你作为leader先自我批评——是不是一直在和稀泥没有建立决策规则？然后分别1对1谈，最后建议事规则。
>
> *教员语录：团结—批评—团结。*

**"产品经理提了一堆需求，技术觉得不靠谱但说不出哪里不对"**

> **缺的不是争论，是调查**。让产品去跟5个真实用户聊，让技术看线上监控数据。双方都带着事实回来时，90%的争论会自动消失。
>
> *教员语录：没有调查，没有发言权。*

### 📈 业务战略 / 职业发展

**"20人AI创业公司，刚拿A轮，大厂发布了类似产品还免费"**

> **形势研判**：大厂做通用方案，你做垂直场景，战场不同。现在是防御期——不全面对抗，集中力量在最强的3个客户场景建根据地。同时找同赛道非直接竞品、被大厂忽视的渠道商做联合。
>
> *教员语录：星星之火，可以燎原。*

**"35岁，技术还行但没特别突出，管理岗竞争激烈，要不要跳槽"**

> **先找主要矛盾**：焦虑的根源是"收入天花板"还是"成长停滞"还是"安全感缺失"？不同矛盾对策完全不同。然后做能力盘点——你的不可替代性在哪？哪些能力是"带得走的"？跳槽只是手段，核心是建自己的壁垒。
>
> *教员语录：外因通过内因起作用。*

## 每次输出你会得到什么

- **问题重定义**：你以为的问题往往不是真正的问题
- **结构化分析**：方法论框架拆解，客观简短直接
- **可执行建议**：具体到"下一步做什么"
- **实践检验**：结论转化为可验证假设，给出验证方法和成败标准
- **批判性反思**：分析的局限、假设的风险、可能的反驳
- **教员语录**：每次输出附一句最贴切的教员原话，画龙点睛

## 使用方式

### 作为 Claude Code / Mira Skill

将 `SKILL.md` 放入你的 Skill 目录即可自动加载。

### 触发方式

不需要记方法论名称，正常说你的问题就行：

- *"帮我分析一下当前的竞争形势"*
- *"团队有分歧怎么办"*
- *"资源不够，怎么排优先级"*
- *"我们应该自研还是外购"*
- *"项目失败了怎么复盘"*

也可以直接指定：*"用矛盾分析法帮我拆一下这个问题"*
